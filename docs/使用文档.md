# java-tracer 使用文档

java-tracer 是一个基于 Java Agent 的**方法调用跟踪记录工具**，可在不修改业务代码的前提下，对指定类和方法进行字节码增强，记录调用链路并在 Web 页面查看。

---

## 一、简介

### 1.1 功能概述

- **无侵入跟踪**：通过 `-javaagent` 挂载，对 JVM 中加载的类进行字节码增强。
- **按需配置**：在 Web 控制台动态设置要跟踪的**类名（cname）**、**方法名（mname）**，以及保留的**链路条数（size）**。
- **系统信息**：查看 ClassLoader/类加载信息、当前线程与堆栈信息。
- **链路查看**：支持列表查看、单条链路详情（JSON/XML），以及两种展示模式（CommonTreeInterceptor / CompressTreeInterceptor）。

### 1.2 项目结构

| 模块 | 说明 |
|------|------|
| `tracer-agent` | Agent 入口（Premain），负责加载 spy、启动内嵌 Jetty、注册字节码转换器 |
| `tracer-spy` | 跟踪逻辑、Web 服务、拦截器、链路存储与展示 |

## 二、获取 Agent 包

### 方式一：直接下载

从仓库获取已构建的 JAR：

- 示例地址：`https://github.com/fengzhongke/java-tracer/blob/master/java-tracer.jar`  
- 下载后保存到本地，例如：`D:/tools/java-tracer.jar`

### 方式二：本地编译

```bash
# 在项目根目录执行
mvn clean package
```

打包完成后，在**项目根目录**下会生成 **`java-tracer.jar`**（由 `tracer-agent` 的 maven-antrun-plugin 复制而来）。

> **说明**：若使用 JDK 8，需在 `tracer-agent/pom.xml` 中配置好 `tools.jar` 的 `systemPath`（指向本机 JDK 的 `lib/tools.jar`），否则可能编译失败。

## 三、启动方式

### 3.1 命令行启动

```bash
java -javaagent:<path>/java-tracer.jar -jar your-app.jar
```

例如：

```bash
java -javaagent:D:/tools/java-tracer.jar -jar myapp.jar
```

### 3.2 自定义 Agent 参数

Agent 支持通过 `premain` 的 `args` 传入配置，格式为 **`key=value`**，多项用 **`:`** 分隔。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `port` | 内嵌 Web 服务端口 | `18902` |
| `sleep` | 启动后延迟多少毫秒再注册 Transformer | `1` |
| `mode` | 运行模式：0=普通方法树，1=线程树，2=压缩线程树 | `0` |

示例：

```bash
# 指定端口 19902，延迟 2 秒
java -javaagent:D:/tools/java-tracer.jar=port=19902:sleep=2000 -jar your-app.jar

# 使用线程模式（mode=1），端口 18902
java -javaagent:D:/tools/java-tracer.jar=port=18902:mode=1 -jar your-app.jar
```

### 3.3 IDE 中启动

在 VM 参数中增加：

```text
-javaagent:<path>/java-tracer.jar
```

需要改端口或模式时：

```text
-javaagent:D:/tools/java-tracer.jar=port=19902:mode=0
```

启动后控制台会看到类似输出：

```text
init trace agent with port [18902] and sleep [1]
pages can be found in http://127.0.0.1:18902
```

## 四、访问 Web 控制台

浏览器打开：

```text
http://127.0.0.1:18902
```

若启动了端口为 `19902`，则访问：

```text
http://127.0.0.1:19902
```

顶部导航有两个入口：

- **index**：跟踪配置与链路列表。
- **class**：系统信息（ClassLoader/类）、include/exclude 配置。

## 五、功能说明

### 5.1 Index 页：跟踪配置与链路

#### 设置要跟踪的类与方法

1. 点击 **「set」** 打开弹窗。
2. 填写：
   - **cname**：要跟踪的类名（全限定名），如 `com.example.service.UserService`。
   - **mname**：方法名，如 `getUser`。
   - **type**：
     - **CommonTreeInterceptor**：完整方法树。
     - **CompressTreeInterceptor**：压缩方法树（相同调用路径合并计数）。
   - **size**：最多保留的链路条数（默认 5，可调大如 20、50）。
3. 点击 **「submit」** 提交。

提交成功后，符合条件的**新发生的调用**会被记录。

#### 查看当前配置

- 当前 **cname、mname、type、size** 会显示在页面上。
- 点击 **「refresh」** 可重新拉取配置和链路列表。

#### 查看链路列表

- 列表中展示：**类型、开始时间、类.方法、耗时(rt)**。
- 点击某行的 **类.方法** 链接，进入该条链路的详情页。

#### 清除跟踪

- 在 set 弹窗中将 **cname / mname 留空** 后提交，即可清除当前跟踪配置（不再记录新链路）。

### 5.2 Trace 页：单条链路详情

- 从 Index 的链路列表点击某条记录进入，或直接访问：  
  `http://127.0.0.1:18902/trace?id=<seed>`
- 可查看该次调用的完整方法树（JSON 或 XML 格式）。

### 5.3 Class 页：系统信息与 include/exclude

#### ClassLoader 与类信息

- 可查看当前 JVM 中按 ClassLoader 划分的类加载情况（通过 `/class/get.xml` 等接口获取 XML 结构）。

#### include / exclude 配置

用于控制**哪些类会被 Agent 增强**（只增强“未在 exclude 且满足 include”的类）：

- **include**：要增强的类名前缀，多个用 **`;`** 分隔。  
  例如：`com.example;org.springframework` 表示只增强以这两个包为前缀的类。
- **exclude**：不增强的类名前缀，多个用 **`;`** 分隔。  
  例如：`com.example.dto;java.` 表示不增强这些前缀的类。

规则说明：

- 先匹配 **exclude**，命中则**不增强**。
- 再匹配 **include**：命中则增强；若 include 为空或未命中，则**不增强**。
- 支持通过系统属性 **`tracer.config.file`** 指定配置文件路径，Agent 启动时会读取该文件作为初始 include/exclude 配置（文件格式见下文）。

## 六、配置文件（可选）

若希望通过文件预设 include/exclude，可指定 JVM 系统属性：

```bash
java -Dtracer.config.file=D:/conf/tracer.conf -javaagent:D:/tools/java-tracer.jar -jar your-app.jar
```

配置文件格式示例：

```text
include:
com.yourcompany.service
com.yourcompany.controller
exclude:
com.yourcompany.dto
java.
javax.
```

- 以 `include:` 开头的行为 include 段，下面每行一个前缀，直到遇到 `exclude:`。
- 以 `exclude:` 开头的行为 exclude 段，下面每行一个前缀。
- 类名匹配使用 **前缀匹配**（`startsWith`）。

## 七、API 摘要（供二次开发/脚本调用）

| 路径 | 说明 |
|------|------|
| `GET /` 或 `/index` | Index 页面 |
| `POST /index/set` | 设置 cname、mname、type、size（form 或 query） |
| `POST /index/get` | 获取当前跟踪配置（cname、mname、type、size、mode） |
| `POST /index/list` | 获取链路列表（seed、类型、类.方法、时间、rt） |
| `GET /trace?id=<seed>` | 链路详情页 |
| `GET /trace/get.json?id=<seed>` | 链路详情 JSON |
| `GET /trace/get.xml?id=<seed>` | 链路详情 XML |
| `GET /class` | Class 配置页 |
| `POST /class/set` | 设置 include、exclude（多个用 `;` 分隔） |
| `POST /class/get` | 获取当前 include、exclude |
| `GET /class/get.xml` | ClassLoader/类信息 XML |
| `GET /thread/get.xml` | 当前所有线程堆栈 XML |

## 八、注意事项

1. **仅增强尚未加载的类**：Agent 在 `premain` 阶段注册的是对新加载类的转换，已加载的类默认不会自动重算。Class 页的「redefine」等能力可在部分场景下配合使用。
2. **性能**：对热点方法做全量跟踪会增加开销，建议在排查问题时使用，并合理设置 **size** 和 include/exclude，缩小增强范围。
3. **端口**：确保 `port` 未被占用；若本机有防火墙，需放行对应端口以便浏览器访问。
4. **JDK 版本**：项目基于 JDK 8 构建；更高版本 JDK 可能需移除或替换 `tools.jar` 相关依赖后再编译。
